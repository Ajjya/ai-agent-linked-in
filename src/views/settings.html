LinkedIn API Setup

üìã LinkedIn Developer Setup Instructions:
Follow these steps to set up your LinkedIn API access.
  1. Create LinkedIn Developer Account:
    • Go to LinkedIn Developers
    • Sign in with your LinkedIn account
    • Click "Create App"
  2. Configure Your App:
    • App name: "MongoDB Content Agent" (or your preference)
    • LinkedIn Page: Select your personal profile or company page
    • Privacy policy URL: Add a simple privacy policy
    • App logo: Upload a logo (optional)
  3. Request API Access:
    • Go to the "Products" tab
    • Request access to "Share on LinkedIn" and "Sign In with LinkedIn"
    • Fill out the use case form explaining automated content sharing
  4. Get Your Credentials:
    • Go to the "Auth" tab
    • Copy your Client ID and Client Secret
    • Add redirect URL: http://localhost:3000/auth/linkedin/callback
‚ö†Ô∏è Important: LinkedIn API access requires approval and may take several days. During development, you can test with your personal account.

Environment Variables Setup

Update your .env file with the following configuration:

# LinkedIn API credentials
LINKEDIN_CLIENT_ID=your_client_id_here
LINKEDIN_CLIENT_SECRET=your_client_secret_here
LINKEDIN_ACCESS_TOKEN=your_access_token_here

# Posting schedule (0=Sunday, 1=Monday, ..., 6=Saturday)
POSTING_DAYS=2,5 # Tuesday and Friday
POSTING_TIME=10:30 # 10:30 AM
TIMEZONE=Europe/Lisbon

# LinkedIn profile settings
LINKEDIN_PROFILE_ID=your_profile_id
LINKEDIN_COMPANY_ID=mongodbinc

OAuth Flow

To get your access token, you'll need to complete the OAuth flow:

  1. Make sure your LinkedIn app is configured with the correct redirect URL
  2. Visit the authorization URL (will be provided in future updates)
  3. Grant permission to your LinkedIn app
  4. Copy the access token to your .env file
üí° Tip: Once you have valid credentials, the system will automatically handle posting to your LinkedIn profile or company page (when you get admin access).

Deployment on Digital Ocean

When you're ready to deploy to Digital Ocean:

  1. Create a Digital Ocean Droplet (Ubuntu recommended)
  2. Install Docker and Docker Compose
  3. Clone your GitHub repository
  4. Set up environment variables on the server
  5. Use the included Docker configuration to deploy
